# ピンポンの記録 仕様書

## 1. 概要

本システムは、卓球の試合結果をローカル環境で記録・参照・分析するための Streamlit アプリケーションである。  
SQLite データベースを用いて試合情報を保存し、次の 3 つの機能を提供する。

- 履歴と編集
- 分析・ダッシュボード
- 試合結果の登録

## 2. 目的

- 試合結果を簡単に登録する
- 相手情報とスコアを一元管理する
- 過去の記録を検索・修正・削除できるようにする
- 戦型別勝率や課題タグの傾向を確認し、今後の練習に活かす

## 3. 対象環境

- 実行形態: ローカル実行
- アプリ基盤: Streamlit
- データ保存先: SQLite
- 主な使用ライブラリ:
  - `streamlit`
  - `pandas`
  - `sqlite3`
  - `json`

## 4. 画面構成

アプリ名は「ピンポンの記録」とする。  
初期表示タブは `履歴と編集` とする。

### 4.1 履歴と編集

過去に登録した試合記録を一覧表示し、詳細確認・修正・削除を行う画面。

#### 表示内容

- 通算成績
  - 勝数
  - 敗数
- 検索条件
  - 対戦相手名の部分一致検索
  - 戦型での絞り込み
- 試合一覧
  - ID
  - 日付
  - 大会名
  - 対戦相手
  - 戦型
  - 自分セット
  - 相手セット
- 試合詳細
  - 大会名
  - 所属チーム
  - 戦型
  - 利き手
  - ラケット
  - フォアラバー
  - バックラバー
  - セットカウント
  - 各ゲームスコア
  - 課題タグ
  - 勝因・敗因メモ

#### 操作

- 一覧の行を 1 件選択すると詳細を表示する
- `この記録を修正する` で編集フォームを開く
- `この記録を削除する` で削除確認を表示する
- 削除確認後に確定すると対象レコードを削除する

### 4.2 分析・ダッシュボード

登録済み試合データを集計して、傾向を可視化する画面。

#### 表示内容

- 戦型別勝率 (%)
  - `play_style != 未選択` のデータを対象に集計
  - 勝敗判定は `my_set_count > opp_set_count`
- 課題タグ集計
  - 集計対象件数を選択可能
  - 選択肢: `5, 10, 20, 50, 100, すべて`
  - 直近試合から `issue_tags` を集計して棒グラフ表示

#### データが空の場合

- 試合データが 1 件もない場合は案内メッセージを表示する

### 4.3 試合結果の登録

新しい試合結果を登録する画面。

#### 入力セクション

- 基本情報
  - 日付
  - 大会名
  - 対戦相手名
  - 所属チーム
- 相手の情報
  - 戦型
  - 利き手
  - フォアラバー
  - バックラバー
- スコア詳細
  - 試合形式: 3 / 5 / 7 ゲームマッチ
  - 各ゲームの自分得点
  - 各ゲームの相手得点
  - 自動計算されたセットカウント
- 振り返り
  - 課題タグ
  - 勝因・敗因 / メモ

## 5. 入力仕様

### 5.1 基本入力項目

| 項目名 | 必須 | 入力形式 | 備考 |
| --- | --- | --- | --- |
| 日付 | 任意 | 日付 | 初期値は当日 |
| 大会名 | 任意 | 文字列 | 空可 |
| 対戦相手名 | 必須 | 文字列 | 未入力時は登録不可 |
| 所属チーム | 任意 | 文字列 | 空可 |
| 戦型 | 任意 | 選択 | 未選択あり |
| 利き手 | 任意 | 選択 | 未選択あり |
| ラケット | 必須 | 選択 | デフォルトはシェーク |
| フォアラバー | 任意 | 選択 | 未選択あり |
| バックラバー | 任意 | 選択 | 未選択あり |
| 試合形式 | 必須 | 3 / 5 / 7 | デフォルトは 5 |
| 各ゲーム得点 | 条件付き必須 | 数値 | 0 から 50 |
| 課題タグ | 任意 | 複数選択 | 複数可 |
| 勝因・敗因 / メモ | 任意 | 複数行文字列 | 空可 |

## 6. バリデーション仕様

### 6.1 必須チェック

- 対戦相手名が未入力の場合はエラーとする

### 6.2 スコアチェック

各ゲームのスコアに対して以下を検証する。

- 入力済みゲームでは、どちらかが 11 点以上に達している必要がある
- `11-x` の決着では、敗者側が 10 点未満である必要がある
- 12 点以上で決着する場合は必ず 2 点差である必要がある
- 試合形式に応じた必要セット数に達していない場合はエラーとする
- 勝敗決定後に余分なゲームスコアが入力されている場合はエラーとする

### 6.3 セット数計算

各ゲームスコアから自動でセット数を算出する。  
`0-0` は未入力ゲームとして扱う。

## 7. データベース仕様

データベースファイル名は `pinpon.db` とする。  
テーブル名は `matches` とする。

### 7.1 テーブル定義

| カラム名 | 型 | 内容 |
| --- | --- | --- |
| id | INTEGER | 主キー、自動採番 |
| match_date | TEXT | 試合日 |
| tournament_name | TEXT | 大会名 |
| opponent_name | TEXT | 対戦相手名 |
| opponent_team | TEXT | 所属チーム |
| play_style | TEXT | 戦型 |
| fore_rubber | TEXT | フォアラバー |
| back_rubber | TEXT | バックラバー |
| dominant_hand | TEXT | 利き手 |
| racket_grip | TEXT | ラケット種別 |
| game_count | INTEGER | 試合形式 |
| my_set_count | INTEGER | 自分のセット数 |
| opp_set_count | INTEGER | 相手のセット数 |
| scores | TEXT | 各ゲームスコアの JSON |
| win_loss_reason | TEXT | 勝因・敗因メモ |
| issue_tags | TEXT | 課題タグの JSON |
| created_at | TEXT | 登録日時 |

### 7.2 JSON保存形式

#### scores

各ゲームスコアを配列で保存する。

例:

```json
[[11, 8], [9, 11], [11, 7], [11, 6], [0, 0]]
```

#### issue_tags

課題タグを文字列配列で保存する。

例:

```json
["サーブミス", "フットワーク"]
```

### 7.3 マイグレーション

起動時に `matches` テーブルへ以下カラムが存在しない場合、追加を試行する。

- `opponent_team`
- `fore_rubber`
- `back_rubber`
- `game_count`

既に存在する場合は例外を無視して継続する。

## 8. 主要処理

### 8.1 起動時処理

- Streamlit ページ設定を行う
- カスタム CSS を適用する
- データベース初期化処理を実行する

### 8.2 新規登録処理

1. 入力値を取得する
2. 対戦相手名の必須チェックを行う
3. スコアバリデーションを行う
4. セット数を自動計算する
5. `matches` テーブルへ INSERT する
6. 登録成功メッセージを表示する
7. 画面を再描画する

### 8.3 編集処理

1. 履歴一覧から対象試合を選択する
2. 修正ボタン押下で編集フォームを開く
3. 初期値入りフォームを表示する
4. 保存時に UPDATE を実行する
5. 編集画面を閉じて再描画する

### 8.4 削除処理

1. 履歴詳細から削除ボタンを押下する
2. 確認メッセージを表示する
3. 確定時に対象レコードを DELETE する
4. 一覧選択状態を初期化して再描画する

### 8.5 分析処理

- 全試合データを読み込む
- 勝敗フラグを生成する
- 戦型別勝率を集計する
- 直近 N 試合または全試合の課題タグ出現数を集計する

## 9. セッション状態

以下の状態を `st.session_state` で管理する。

- `show_edit_{id}`
  - 対象レコードの編集フォーム表示状態
- `show_del_{id}`
  - 対象レコードの削除確認表示状態
- `history_table`
  - 履歴一覧の選択状態

## 10. エラー時の挙動

- データ読み込み失敗時はエラーメッセージを表示する
- 登録・更新・削除処理失敗時はエラーメッセージを表示する
- スコア不整合時はエラー内容を複数件表示する

## 11. 制約事項

- 認証機能はない
- ユーザー管理機能はない
- 複数端末同期はない
- エクスポート・インポート機能はない
- データ保存先はローカル SQLite 固定である
